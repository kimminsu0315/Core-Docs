## 문서 안내

이 문서에서 시스템 측은 AMMR 외부의 운영 시스템(외부 발주·운영 Dashboard·통신 게이트웨이 등 일체)을 가리킵니다.

이 문서에 등장하는 어휘와 형식은 두 가지로 나뉩니다.

- **고정 항목**: 정보 핵심과 직결된 어휘·형식 (Unit 식별값 형식 `투입코드_유닛번호`, Log 분류 어휘, 본문에 직접 적힌 라벨 어휘 등) — 시스템 측과 일관되어야 하므로 본문에 적힌 그대로 적용해 주세요.
- **자유 항목**: 표기 방식만 자유 (Sensor 표시 기호·비어 있음 표기·버튼 형태·색·아이콘·폰트·배치 등) — 본문에 등장하는 예시는 참고용이며, 내용이 통한다면 업체에서 자유롭게 구성해 주셔도 좋습니다.

## 1. 태블릿 UI 개요

AMMR 1대당 태블릿 1대가 본체에 Mount되며, 항상 켜져 있고 한 계정으로 자동 로그인된 상태에서 운영됩니다. 이 태블릿은 담당자가 현장에서 AMMR의 현재 상태를 확인하고 필요한 설정을 조정하기 위한 화면입니다.

이 문서에서는 다음 항목을 제안드립니다.

- 모든 화면 공통으로 항상 보여주셨으면 하는 AMMR 정보 (상단 고정 영역)
- 운영 중 화면 4종 (적재 상태 / 배정 상태 / 설정 / Log)
- 각 화면에 표시되었으면 하는 정보 항목
- 설정 화면의 입력 항목과 동작
- 참고용 UI 모형 4장 (화면별)

---

## 2. 권장 화면 구성

태블릿 화면은 **왼쪽 Sidebar + 본문 영역** 구조로 구성을 권장드립니다.

- **왼쪽 Sidebar**: 화면 전환 메뉴 (적재 상태 / 배정 상태 / 설정 / Log 4개 항목). 모든 화면에서 항상 표시되며, Sidebar 항목을 누르면 본문 영역이 해당 화면으로 전환되는 동작을 부탁드립니다.
- **상단 정보 영역**: AMMR의 핵심 정보를 표시하는 영역으로, 어느 화면에 있더라도 항상 같은 위치에 고정 표시됩니다.
- **본문 영역**: Sidebar에서 선택된 화면이 표시되는 영역입니다.

화면 디자인·색상·폰트·세부 배치는 업체 자유 결정 영역입니다.

## 3. 상단 정보 영역 (모든 화면 공통, 항상 고정 표시)

다음 7개 항목이 항상 표시되도록 구성을 부탁드립니다.

※ 상단 정보 항목

| 항목         | 내용                                                                       |
|--------------|----------------------------------------------------------------------------|
| AMMR ID      | AMMR 고유 식별자 (예: `ammr-001`)                                          |
| HW 상태      | AMMR이 보고하는 현재 상태                                                  |
| 시스템 상태  | 시스템이 판단한 AMMR 상태                                                  |
| 작업 슬롯    | 현재 동작 중인 Slot 번호 (Job 수행 중일 때만 표시)                        |
| 위치         | 현재 위치 좌표(x, y)와 방향각(a) (예: `12.4, 8.7, 90°`)                    |
| Battery       | 현재 충전량 (%)                                                            |
| 시스템 연결  | 시스템과의 연결 상태 표시 + 연결/해제 동작 버튼                            |

각 항목의 구체 표기 형식·기호·아이콘은 업체 자유 결정 영역입니다. 내용이 식별 가능한 형태로만 구성되면 됩니다. 단, 시스템 상태 값은 고정 항목입니다 — 아래 표기 값 안내를 참고해 주세요.

**HW 상태·시스템 상태 표기 값:**

Job 수행 중에는 현재 수행 중인 동작 단위를, Job이 없을 때는 운영 상태를 표시해 주세요.

| 분류              | 표기 값                                  |
|-------------------|------------------------------------------|
| Job 수행 중      | `Move` / `Pickup` / `Dropoff` / `Charge` |
| Job 없음·운영 중 | `대기` / `충전 중` / `저전력` / `장애`   |

HW 상태 칸의 표기는 위 값을 예시로 참고한 업체 자유 결정 영역입니다 — 동작 단위가 구분되고 운영 상태가 식별 가능한 형태이면 됩니다. 시스템 상태 칸은 고정 항목입니다 — 시스템이 응답으로 내려주는 표시 텍스트(위 표의 값)를 가공 없이 그대로 표시해 주세요.

HW 상태와 시스템 상태는 보통 같은 값이지만, 통신 단절이나 시스템 측 판단 차이가 발생하면 서로 다르게 표시될 수 있어 두 값을 따로 표시합니다.

**시스템 연결 상태 + 연결/해제 버튼:**

AMMR이 장애·오류 발생 시 담당자가 시스템 연결을 끊고 조치 후 재연결할 수 있도록 연결/해제 버튼을 함께 제공해 주세요. 연결 중일 때와 끊김 상태일 때를 구분할 수 있는 시각 표시(예: 색상·아이콘·기호 등)와 상태에 따라 동작 가능한 버튼(예: `[연결]` / `[해제]` 또는 Toggle 등)을 함께 두는 형태가 권장됩니다. 구체 표기·버튼 형태는 업체 자유 결정 영역입니다.

**시스템 연결이 끊긴 동안의 표시 방식:**

이 문서에서 다루는 표시값은 출처에 따라 권위가 둘로 나뉩니다 — AMMR 자체에서 보고되는 값(HW 상태·위치·Battery·Sensor 등 AMMR 권위)과 시스템이 응답으로 내려주는 값(시스템 상태·작업 슬롯·Slot의 정보·배정 상태 화면의 표시값 등 시스템 권위). 시스템 연결이 끊긴 동안에는 두 권위 출처에 따라 표시 방식이 달라집니다. AMMR 권위 값은 시스템 연결과 무관하게 마지막 수신값을 그대로 유지하고, 시스템 권위 값은 시스템의 판단이 필요한 값이므로 빈값으로 표시됩니다. 이 일반 원칙은 이 문서의 적재 상태·배정 상태 화면 표시값에 동일하게 적용됩니다.

이 상단 정보 영역에서의 항목별 적용은 다음과 같습니다.

- **HW 상태·위치·Battery**: AMMR 자체에서 보고되는 값이므로 마지막 수신값을 그대로 유지합니다.
- **시스템 상태·작업 슬롯**: 시스템의 판단이 필요한 값이므로 빈값으로 표시합니다 (예: 항목 칸을 빈 텍스트로 두거나 회색 계열로 표시).

시스템 연결 항목의 끊김 표시(예: ○ + `[연결]`)가 단절 상태를 표면화하는 역할을 합니다. 단절 상태에서 다른 항목 값에 별도의 시각 구분(예: 흐리게·기호 부착 등)을 추가하실지 여부는 업체 자유 결정 영역입니다.

## 4. 적재 상태 화면

AMMR의 6개 Slot 각각의 현재 적재 상태(Sensor·시스템 상태·Unit 정보)를 격자 형태로 보여주는 화면입니다. Slot당 한 칸의 셀이며, 각 셀에는 다음 정보가 함께 표시됩니다.

※ Slot 셀 표시 정보

| 항목       | 내용                                                                  |
|------------|-----------------------------------------------------------------------|
| Slot 번호  | 1 ~ 6                                                                 |
| Sensor       | AMMR HW가 보고하는 Slot 점유 Sensor 값 (점유 / 비점유 구분 가능 표기)   |
| 시스템 상태| 시스템이 판단한 Slot 상태 (정상 / 정합성 이상)                                    |
| 정보       | Slot에 적재된 Unit 식별값. 비어 있을 때는 비어 있음을 알 수 있는 표기 |

각 항목의 표기 방법은 업체 자유 결정 영역입니다. 예를 들어 Sensor는 `●/○`·`ON/OFF`·아이콘 등, 비어 있음은 `(비어 있음)`·`Empty`·`--` 등 내용이 통하는 어떤 형태로든 표시 가능합니다.

**시스템 상태 표기 방식:**

시스템 상태는 시스템이 내려주는 텍스트 값을 그대로 표시합니다. 표시 텍스트 자체를 가공·변형하지 않으며, 상태 구분을 돕는 보조 시각 표시(색·아이콘 등)를 함께 두는 것은 업체 자유 결정 영역입니다.

**Slot 정합성 이상의 의미:**

Slot이 "정합성 이상"으로 표시된다는 것은 시스템이 해당 Slot을 사용 보류로 판단했다는 의미입니다. 예를 들어 Sensor는 점유로 잡혀 있는데 시스템 쪽에서는 적재된 Unit 정보가 없는 경우(사람이 임의로 Unit을 올려놓았으나 시스템이 인지하지 못한 상황 등)에 발생합니다. 이 경우 Sensor는 점유 표시이지만 정보 칸은 비어 있음을 나타내는 표기가 됩니다.

**Unit 식별값과 스티커:**

Unit 식별값은 `투입코드_유닛번호` 형태로 표시됩니다. 예시는 `26SF03002-001_001`. 투입코드 부분은 시스템이 외부로부터 받은 발주 단위 코드이고, 언더바 뒤 3자리는 해당 투입코드 안에서 부여된 Unit 일련번호입니다.

Unit 실물에는 이 식별값을 사람이 눈으로 대조할 수 있는 라벨과, 시스템이 기계 입력으로 받을 수 있는 QR 코드가 함께 부착된 스티커가 붙어 있습니다. 라벨에는 이 식별값(`투입코드_유닛번호`)이 사람이 읽을 수 있는 형태로 적혀 있고, QR 코드에는 시스템 권위 식별값인 Unit UUID가 인코딩되어 있습니다.

이 화면에 표시되는 식별값은 담당자가 현장에서 스티커 라벨과 직접 대조해 확인하기 위한 정보입니다. UUID는 적재 정보를 수동으로 입력해야 하는 경우 설정 화면의 적재 정보 관리 영역에서 사용되며, 자세한 동작은 이 문서 뒤쪽 설정 화면과 적재 정보 재로드 동작 명세 부분을 참고해 주세요.

**시스템 연결이 끊긴 동안의 표시 방식:**

표시값은 권위 출처에 따라 단절 동안 표시 방식이 달라집니다.

- **Slot 번호**: 고정 항목이므로 항상 표시됩니다.
- **Sensor**: AMMR 자체에서 보고되는 값이므로 마지막 수신값을 그대로 유지합니다.
- **시스템 상태 · 정보**: 시스템이 응답으로 내려주는 값이므로 단절 동안 빈값으로 표시합니다.

**갱신 방식:**

이 화면의 모든 정보는 시스템이 자동으로 갱신합니다. 담당자가 별도 새로고침 동작을 하지 않아도 됩니다.

## 5. 배정 상태 화면

각 Slot에 매칭된 Job 정보를 보여주는 화면입니다. 6개 Slot 모두 항목으로 표시되며, Job이 배정되어 있지 않거나 Slot이 비어 있는 경우 또는 Slot 정합성 이상 상태인 경우에는 그 상태를 알 수 있는 표기로 표시됩니다.

※ Slot별 표시 정보 (Job이 배정된 Slot에 한함)

| 항목         | 내용                                                                                                       |
|--------------|------------------------------------------------------------------------------------------------------------|
| Unit 식별값  | 해당 Slot에 적재된 Unit 식별값 (적재 상태 화면과 동일 형식 `투입코드_유닛번호`)                            |
| 모델         | Unit에 담긴 제품의 모델 코드                                                                               |
| 버전         | 모델의 버전 표기                                                                                           |
| Tray 단      | Unit을 구성하는 Tray 단 수                                                                                 |
| 수량         | Unit 안 제품 개수                                                                                          |
| 출발지       | 이 Unit을 어디서 Pickup했거나 Pickup할 위치 (공정명 + Slot 라벨. 예: `세척 WIP 슬롯 A1`·`CNC 1 작업대 슬롯 A2`)  |
| 도착지       | 이 Unit을 어디로 가져갈 위치 (공정명 + Slot 라벨. 예: `되담기 WIP 슬롯 B3`·`DP WIP 슬롯 B1`)                 |

표기 형식·배치는 업체 자유 결정 영역입니다.

**비어 있는 Slot · Job 미배정 Slot · Slot 정합성 이상의 표기:**

비어 있는 Slot은 비어 있음을 알 수 있는 표기(예: `(비어 있음)`), 적재되어 있으나 Job이 아직 배정되지 않은 Slot은 작업 배정 대기를 알 수 있는 표기(예: `(작업 배정 대기)`), Slot 정합성 이상은 정합성 이상 상태를 알 수 있는 표기(예: `(슬롯 정합성 이상)`)가 권장됩니다.

작업 배정 대기 상태는 두 가지 상황에서 발생합니다. 첫째, 적재 정보는 들어왔지만 시스템이 아직 매칭된 Job을 결정하기 전인 일시적인 상태입니다. 둘째, 시스템 연결이 끊긴 동안 Slot의 시스템 권위 정보 일체가 빈값으로 표시되는 상태입니다.

**시스템 연결이 끊긴 동안의 표시 방식:**

이 화면의 표시값(Unit 식별값·모델·버전·Tray·수량·출발·도착) 일체는 모두 시스템이 응답으로 내려주는 값이므로, 시스템 연결이 끊긴 동안에는 Slot 번호 외 표시값이 모두 빈값으로 표시됩니다. 빈 칸은 작업 배정 대기 표기로 두어도 좋습니다.

**우선순위 표기에 대하여:**

Job의 우선순위는 시스템 측이 판단해 Slot에 할당합니다. 이 화면은 순번이 아니라 Slot 번호 기준으로 표시되며, 별도의 우선순위 번호는 표시하지 않습니다.

**갱신 방식:**

이 화면의 모든 정보 역시 시스템이 자동으로 갱신합니다.

## 6. 설정 화면

설정 화면은 다음 3개 영역으로 구성됩니다 — **Battery·충전 운영**, **시스템 접속**, **적재 정보 관리**.

각 영역의 입력 항목은 영역별 동작 방식으로 시스템 또는 AMMR에 반영됩니다.

- **Battery·충전 운영 영역**: 영역 안 입력 항목을 모은 뒤 [저장] 버튼으로 일괄 반영합니다. AMMR 자체에서 사용하는 값이므로 시스템 연결 상태와 무관하게 항상 활성화됩니다.
- **시스템 접속 영역**: 영역 안 입력 항목을 모은 뒤 [저장] 버튼으로 일괄 반영합니다. 이 영역은 시스템 연결이 끊긴 상태에서만 활성화됩니다 (운영 중 시스템 접속 정보를 변경하면 통신에 영향을 주는 것을 피하기 위함입니다).
- **적재 정보 관리 영역**: Slot별 입력 박스로 Unit UUID를 수동 입력하고 [Slot별 적재 정보 재로드] 버튼으로 시스템에 일괄 전달하는 방식입니다. 시스템 연결 상태에 따라 입력 박스와 버튼의 활성 방식이 반대로 동작하며, 자세한 내용은 이 영역 항목 설명에서 다룹니다.

이 화면 3개 영역의 입력 값은 모두 AMMR 내부에 저장되어 AMMR 재시작 후에도 유지됩니다.

각 영역의 버튼 명칭·형태·배치는 업체 자유 결정 영역입니다.

### 6.1 Battery·충전 운영

※ Battery·충전 운영 입력 항목

| 항목                  | 초기값 | 비고                                                                   |
|-----------------------|--------|------------------------------------------------------------------------|
| 충전 스테이션 번호    | -      | AMMR이 충전 시 이동할 스테이션 번호 (자율 충전·시스템 충전 지시 공통) |
| Battery 저전력 임계치 | 20%    | AMMR이 자율 충전 진입을 판단하는 값                                    |
| Battery 대기 임계치   | 80%    | AMMR이 충전을 마쳤다고 판단하는 값                                     |

충전 스테이션 번호는 단순 정수로 입력받습니다 (예: `1`, `2`, `3`).

**이 영역 안내 문구 (화면에 함께 표시):**

> 충전 스테이션 번호는 AMMR이 충전 시 이동할 스테이션 위치이고(자율 충전과 시스템의 충전 지시 모두 이 스테이션을 사용합니다), Battery 저전력·대기 임계치는 AMMR이 지정된 충전 스테이션으로 스스로 충전·복귀 동작을 결정하는 기준값입니다. 출고 시 각각 20%·80%로 설정되어 있으며, 운영 중 현장 상황에 맞게 설정 화면에서 변경하실 수 있습니다. 변경하신 값은 [저장] 버튼을 누르신 시점에 적용됩니다.

### 6.2 시스템 접속

※ 시스템 접속 입력 항목

| 항목             | 초기값 | 비고                                |
|------------------|--------|-------------------------------------|
| 시스템 Host IP | -      | AMMR이 시스템에 접속할 IP 주소      |
| 시스템 포트      | -      | 시스템 접속 포트 번호               |
| MQTT 접속 ID | -      | 시스템 측 MQTT Broker 접속 자격증명 |
| MQTT 접속 비밀번호   | -      | 시스템 측 MQTT Broker 접속 자격증명 |
| 일괄 보고 주기 | 60초 | AMMR이 운영 상태값 전체를 주기적으로 자동 보고하는 간격 |

위 접속 4개 항목을 입력하신 뒤 [저장] 버튼으로 일괄 적용됩니다. 접속 4개 항목은 시스템 연결이 끊긴 상태에서만 활성화되며, 일괄 보고 주기는 연결 상태와 무관하게 항상 변경 가능합니다.

### 6.3 적재 정보 관리

이 영역은 **Slot별 Unit 정보 입력 박스 6개 + Slot별 [초기화] 버튼 6개**와 **Slot별 적재 정보 재로드 버튼 1개**로 구성합니다.

이 영역은 시스템 연결이 끊긴 동안 담당자가 Slot별 적재 정보를 수동으로 입력해 두고, 시스템 재연결 후 [Slot별 적재 정보 재로드] 버튼으로 일괄 전달해 Slot 정합성 이상을 풀기 위한 영역입니다. 정상 운영(시스템 연결 유효 + AMMR Job 수행) 중에는 시스템이 Slot의 적재 정보를 자동으로 채우므로, 이 영역에 담당자가 별도로 개입할 필요가 없습니다 (자세한 동작은 이 문서 뒤쪽 적재 정보 재로드 동작 명세 부분을 참고해 주세요).

**Slot별 Unit 정보 입력 박스:**

- Slot 1 ~ 6 각각에 텍스트 입력 박스를 1개씩 배치합니다.
- 이 입력 박스에 들어가는 값은 **Unit UUID** (시스템이 부여하는 시스템 권위 식별값)입니다. 이 UUID는 적재 상태 화면에 표시되는 `투입코드_유닛번호` 라벨과는 별도의 값입니다. 라벨은 담당자가 Slot의 Unit을 시각적으로 식별하기 위한 사람 읽기용 표기이고, UUID는 시스템이 Unit을 권위적으로 인지하기 위한 시스템 입력용 식별값입니다.
- 적재된 Slot의 입력 박스에는 마지막 시스템 응답 또는 [재로드] 시점의 Unit UUID가 기본값으로 표시됩니다. 이 값은 AMMR 내부에 저장되어 재시작 후에도 유지되고, 정상 운영 중 시스템 자동 갱신되는 곳에는 갱신값으로 덮어 씌워집니다. 비어 있는 Slot의 입력 박스는 빈 상태로 표시됩니다.
- 입력 박스를 터치하면 활성화되어 가상 키보드가 표시되고, 담당자가 직접 값을 입력할 수 있습니다.
- 향후 QR 리딩을 통한 입력도 지원되도록 구성을 부탁드립니다 (스티커에 부착된 QR 코드를 스캐너로 읽어 해당 Slot의 입력 박스 값으로 채워 넣는 방식. QR 코드에는 이 UUID가 인코딩되어 있습니다).

**Slot별 [초기화] 버튼:**

- 각 Slot 입력 박스 우측에 [초기화] 버튼을 1개씩 배치합니다.
- 이 버튼은 해당 Slot 입력 박스의 값을 비우는 동작입니다. AMMR 내부 저장값도 즉시 비워집니다.
- 시스템에 단독 전달되지는 않습니다 — 입력 박스 값 변경분은 [Slot별 적재 정보 재로드] 버튼을 눌러야 시스템에 일괄 전달됩니다.

**Slot별 적재 정보 재로드 버튼:**

이 버튼은 AMMR이 6Slot 전체의 Sensor 값 + 입력된 UUID(있으면)를 시스템에 일괄 전달하는 동작을 Trigger합니다. 시스템은 받은 정보를 토대로 Slot별 Unit UUID·Unit 정보·Job 배정을 응답으로 일괄 내려주며, 이 응답으로 적재 상태 화면과 배정 상태 화면이 일괄 갱신됩니다.

**이 영역 안내 문구 (화면에 함께 표시):**

> Slot별 입력 박스를 터치하면 가상 키보드가 표시되어 값을 직접 입력하실 수 있습니다. 향후 QR 리딩으로도 입력 가능합니다. Slot별 [초기화] 버튼을 누르면 해당 Slot 값이 비워집니다. [Slot별 적재 정보 재로드] 버튼을 누르시면 6Slot 전체의 Sensor 값과 입력된 UUID가 시스템에 일괄 전달되어 Slot 정합성 이상이 자연스럽게 풀립니다. 입력 박스와 [초기화] 버튼은 시스템 연결이 끊긴 상태에서만, [재로드] 버튼은 시스템 연결이 유효한 동안에만 활성화됩니다. 이 화면 입력 값은 AMMR 재시작 후에도 유지됩니다. 자세한 동작은 이 문서 뒤쪽 적재 정보 재로드 동작 명세 부분을 참고해 주세요.

**활성화 조건:**

이 영역의 활성화 방식은 입력 박스·[초기화] 버튼과 [Slot별 적재 정보 재로드] 버튼이 서로 반대입니다.

- **Slot별 입력 박스·[초기화] 버튼**: 시스템 연결이 끊긴 상태에서만 활성화됩니다. 정상 운영 시에는 시스템이 적재 정보를 자동으로 채우므로 수동 입력이 불필요합니다.
- **[Slot별 적재 정보 재로드] 버튼**: 시스템 연결이 유효한 동안에만 활성화됩니다. 시스템에 정보를 전달하는 동작이라 연결이 끊긴 동안에는 동작할 수 없습니다.

따라서 일반적인 사용 흐름은 다음과 같습니다 — 시스템 연결이 끊긴 동안 Slot 입력 박스에 UUID를 입력해 두고 → 시스템 재연결 → [Slot별 적재 정보 재로드] 버튼을 눌러 입력값을 시스템에 일괄 전달.

## 7. 적재 정보 재로드 동작 명세

이 장은 적재 정보가 시스템과 AMMR 사이에서 어떻게 흐르는지, 그리고 담당자가 수동으로 개입해야 하는 상황에서 [Slot별 적재 정보 재로드] 버튼이 어떻게 동작하는지를 명세합니다.

**정상 운영 시 적재 정보 흐름:**

정상 운영(시스템 연결 유효 + AMMR Job 수행) 중에는 AMMR Slot의 적재 정보를 시스템이 자동으로 채워 줍니다. AMMR이 Pickup 또는 Dropoff 동작을 수행하면 Slot의 Sensor 값 변동이 시스템에 보고되고, 시스템은 해당 Slot의 Unit UUID와 Unit 정보(모델·버전·Tray·수량·Job)를 응답으로 내려줍니다. 담당자가 설정 화면의 적재 정보 관리 영역에 별도로 개입할 필요는 없습니다.

**수동 입력이 필요한 상황:**

시스템 연결이 끊긴 상태에서 담당자가 현장에서 AMMR Slot의 Unit을 임의로 가져가거나 새로 올려놓는 상황이 발생할 수 있습니다. 시스템 측은 단절 동안 AMMR Slot의 변동을 인지하지 못하므로, 재연결 직후 적재된 Slot을 Slot 정합성 이상 상태로 표시해 둡니다. 담당자는 시스템 연결이 끊긴 동안 Slot 정합성 이상을 미리 풀어 두기 위해 설정 화면의 적재 정보 관리 영역에서 Slot별 입력 박스에 Unit UUID를 직접 입력합니다. UUID 입력은 가상 키보드 또는 QR 리딩(스티커 QR 코드에 인코딩된 UUID를 스캐너로 읽어 입력)으로 가능합니다.

**[Slot별 적재 정보 재로드] 일괄 전달 동작:**

담당자가 [Slot별 적재 정보 재로드] 버튼을 누르면 AMMR은 6Slot 전체의 Sensor 값 + 입력된 UUID(있으면)를 시스템에 일괄 전달합니다. 시스템은 받은 정보를 토대로 Slot별 응답을 일괄 내려줍니다. 응답에는 Slot별 Unit UUID·Unit 정보(모델·버전·Tray·수량) + Job 배정 정보(출발·도착)가 포함되며, 이 응답으로 적재 상태 화면과 배정 상태 화면이 일괄 갱신됩니다.

**응답 없음 시 처리:**

[Slot별 적재 정보 재로드] 버튼을 누른 뒤 3초 안에 시스템 응답이 오지 않으면 실패로 판단해 메시지 박스로 안내하고, 담당자가 다시 시도하실 수 있습니다.

**버튼 활성화 조건:**

[Slot별 적재 정보 재로드] 버튼은 시스템 연결이 유효한 동안에만 활성화됩니다. 시스템 연결이 끊긴 상태에서는 비활성 표시됩니다 (시스템에 정보를 전달하는 동작이라 연결이 끊긴 동안에는 동작할 수 없습니다). 반대로 Slot별 Unit 정보 입력 박스는 시스템 연결이 끊긴 상태에서만 활성화됩니다 (정상 운영 시에는 시스템이 자동으로 채우므로 수동 입력이 불필요합니다).

**응답 종류:**

시스템 응답은 Slot별로 다음 중 하나입니다.

- **정상 응답**: Unit UUID·Unit 정보·Job 배정이 모두 정상 응답되어 화면에 표시됩니다.
- **빈 Slot 응답**: Sensor가 비점유인 Slot은 빈 Slot으로 응답되어 적재 상태 화면의 정보 칸이 비어 있음으로 표시됩니다.
- **Slot 정합성 이상 응답**: 시스템이 입력된 UUID를 인지하지 못하거나, Slot의 정합성(Sensor 값과 UUID 매칭)이 어긋나거나, 적재된 Unit이 갈 수 없는 공정으로 들어온 것으로 판단되면 Slot 정합성 이상으로 응답되어 적재 상태 화면과 배정 상태 화면에 Slot 정합성 이상으로 표시됩니다.

Slot 정합성 이상으로 응답된 Slot은 담당자가 입력 박스 값을 재확인하거나 Slot 실물을 확인하여 다시 [재로드]를 시도할 수 있습니다.

## 8. Log 화면

AMMR에서 발생한 주요 Event를 시간순으로 보여주는 화면입니다. 이 화면의 Log는 AMMR이 자체적으로 기록한 것으로, 시스템 측은 이 화면에 개입하지 않습니다. 담당자가 현장에서 발생한 상황을 점검할 수 있도록 구성합니다.

**기록 분류:**

다음 3종 분류를 한 화면에 통합 시간순으로 표시합니다. 분류 어휘는 시스템 측 Log 분류와 일관되어야 하므로 본문에 적힌 그대로 적용해 주세요.

- **AMMR 변경**: AMMR 자체에서 발생한 상태·구성 변화 (HW 상태 전이, Slot 적재 정보 변동 등)
- **Core 통신**: 시스템과 주고받은 통신 Event (Job 지시 수신, 연결·해제 등)
- **설정 변경**: 설정 화면에서 사용자가 변경한 항목 (임계치 변경, 접속 정보 변경 등)

**표시 컬럼:**

| 컬럼 | 내용                                                          |
|------|---------------------------------------------------------------|
| 시간 | Event 발생 시각 (날짜·시각 표기 형식은 업체 자유 결정 영역)  |
| 분류 | 위 3종 분류 중 하나 (AMMR 변경 / Core 통신 / 설정 변경)       |
| 내용 | Event 요약 문구                                              |

**분류 필터:**

화면 상단에 분류 필터(전체 / AMMR 변경 / Core 통신 / 설정 변경)를 제공합니다. 선택된 분류에 해당하는 항목만 리스트에 표시됩니다. 필터 UI 형태(버튼·Dropdown·Toggle 등)는 업체 자유 결정 영역입니다.

**갱신 방식:**

이 화면의 모든 정보는 AMMR이 자체적으로 기록한 Event를 시간순으로 자동 표시합니다.

## 9. 참고용 UI 모형

아래 모형 4장은 참고용입니다. 어떤 정보가 어떤 영역에 배치되면 좋을지 보여드리기 위해 박스 구성과 Slot·표 배치를 시각화했습니다. **실제 화면의 디자인·색상·폰트·세부 Component는 업체 자유 결정 영역**입니다. 모형 안의 표기 예시(상태 문구·라벨 형식·UUID 예시 등)도 모두 참고용입니다.

각 모형은 외곽 박스 안에 세 영역(상단 정보 영역 / Sidebar / 본문)으로 분리해서 표시했습니다. 실제 화면에서는 상단 정보 영역이 화면 상단에 고정되고, Sidebar가 화면 좌측에 고정되며, 본문이 나머지 영역에 배치됩니다. Sidebar의 `>` 표시는 현재 선택된 화면을 가리키는 표시 예시입니다.

본문 길이가 한 화면을 넘기면(특히 설정 화면) 본문 영역 내부 스크롤로 보이는 방식으로 구성합니다.

모형 안 분류 라벨(예: Log 화면의 `Core 통신`)은 Log 분류 어휘로 굳어진 표기이며, 본문 어휘 "시스템"과는 별도로 둡니다.

이 모형들은 ASCII·박스 문자(─│┌)·기호(●○·°)를 1폭으로, 한글을 2폭으로 두는 가시폭 가정으로 정렬되어 있습니다. D2Coding 같은 한글 Monospace 폰트로 보시면 박스가 정확히 정합되어 보입니다. 그 외 폰트에서는 일부 기호·박스 문자의 폭이 달라 Slot 박스 모서리가 미세하게 어긋나 보일 수 있으나, 정보 배치 의도는 동일합니다.

모형 안 `시스템 ●` 표기는 색으로 시스템 상태를 구분합니다. 정상·시스템 연결 끊김 시는 상단 정보 영역의 시스템 연결 표기와 같은 색으로, Slot 정합성 이상 시는 빨강 계열로 표시해 주세요. 이 모형은 ASCII 표기이므로 색 표기는 없습니다.

### 9.1 적재 상태 화면 모형

```
┌────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│  ammr-001  │ HW: Move  시스템: Move  작업 Slot: 1  위치: 12.4, 8.7, 90° Battery: 78% 시스템 연결: ● [해제] │
├────────────┬───────────────────────────────────────────────────────────────────────────────────────────────┤
│ > 적재 상태│ ┌─ Slot 1 ────────────────────┐┌─ Slot 2 ────────────────────┐┌─ Slot 3 ────────────────────┐ │
│   배정 상태│ │ Sensor ●  시스템 ●          ││ Sensor ○  시스템 ●          ││ Sensor ●  시스템 ●          │ │
│   설정     │ │ 정보 26SF03002-001_001      ││ 정보 (비어 있음)            ││ 정보 26AL05011-002_003      │ │
│   Log      │ └─────────────────────────────┘└─────────────────────────────┘└─────────────────────────────┘ │
│            │ ┌─ Slot 4 ────────────────────┐┌─ Slot 5 ────────────────────┐┌─ Slot 6 ────────────────────┐ │
│            │ │ Sensor ○  시스템 ●          ││ Sensor ●  시스템 ●          ││ Sensor ○  시스템 ●          │ │
│            │ │ 정보 (비어 있음)            ││ 정보 (비어 있음)            ││ 정보 (비어 있음)            │ │
│            │ └─────────────────────────────┘└─────────────────────────────┘└─────────────────────────────┘ │
│            │                                                                                               │
└────────────┴───────────────────────────────────────────────────────────────────────────────────────────────┘
```

위 모형의 Slot 5는 Sensor는 점유로 잡혀 있으나 시스템에는 Unit 정보가 없는 정합성 불일치 상황의 예시입니다.

### 9.2 배정 상태 화면 모형

```
┌────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│  ammr-001  │ HW: Move  시스템: Move  작업 Slot: 1  위치: 12.4, 8.7, 90° Battery: 78% 시스템 연결: ● [해제] │
├────────────┬───────────────────────────────────────────────────────────────────────────────────────────────┤
│   적재 상태│ Slot 1:  Unit 26SF03002-001_001                                                               │
│ > 배정 상태│          모델 H8-MAIN · 버전 KM70 · Tray 5단 · 수량 16개                                      │
│   설정     │          출발 CNC 1 작업대 슬롯 A2 -> 도착 되담기 WIP 슬롯 B3                                 │
│   Log      │ Slot 2:  (비어 있음)                                                                          │
│            │ Slot 3:  Unit 26AL05011-002_003                                                               │
│            │          모델 H7-SIDE · 버전 KM70 · Tray 3단 · 수량 4개                                       │
│            │          출발 세척 WIP 슬롯 A1 -> 도착 되담기 WIP 슬롯 B4                                     │
│            │ Slot 4:  (비어 있음)                                                                          │
│            │ Slot 5:  (슬롯 정합성 이상)                                                                   │
│            │ Slot 6:  Unit 26AL05011-002_005                                                               │
│            │          (작업 배정 대기)                                                                     │
│            │                                                                                               │
└────────────┴───────────────────────────────────────────────────────────────────────────────────────────────┘
```

각 Slot 행은 Unit 식별값(`투입코드_유닛번호`) · 모델·버전·Tray·수량 · 출발 -> 도착 3행으로 구성됩니다. 비어 있는 Slot은 `(비어 있음)`, Slot 정합성 이상 상태는 `(슬롯 정합성 이상)`, 적재되어 있으나 Job이 배정되지 않은 Slot은 `(작업 배정 대기)` 표기로 모형에 넣었습니다. 위 모형의 Slot 6이 작업 배정 대기 상태의 예시입니다.

WIP·작업대 위치 모두 이 모형에 표기된 형태로 `되담기 WIP 슬롯 B3`·`CNC 1 작업대 슬롯 A2`처럼 (공정명 + Slot 라벨) 표기됩니다.

이 모형은 시스템 연결이 유효한 상태(상단 시스템 연결 ●)를 가정해 그렸습니다. 이 가정에 따라 영역별 활성 부분은 다음과 같습니다.

- **Battery·충전 운영 영역**: AMMR 자체 값이므로 시스템 연결 상태와 무관하게 항상 활성. 모형 상태에서도 활성.
- **시스템 접속 영역**: 접속 4개 항목은 시스템 연결이 끊긴 상태에서만 활성이라 모형 상태(연결 유효)에서는 비활성. 일괄 보고 주기는 연결 상태와 무관하게 항상 활성.
- **적재 정보 관리 영역**: Slot별 입력 박스·[초기화] 버튼은 시스템 연결이 끊긴 상태에서만 활성이라 모형 상태에서는 비활성, [Slot별 적재 정보 재로드] 버튼은 시스템 연결이 유효한 동안만 활성이라 모형 상태에서는 활성.
