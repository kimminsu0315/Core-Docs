## 문서 안내

이 문서에서 시스템 측은 AMMR 외부의 운영 시스템(외부 발주·운영 Dashboard·통신 게이트웨이 등 일체)을 가리킵니다.

이 문서는 태블릿 화면을 어떻게 구성하면 좋을지 제안드리는 문서입니다. 화면 디자인·색·폰트·아이콘·배치·버튼 형태·표기 기호는 모두 업체에서 자유롭게 정해 주시면 되고, 본문에 나오는 예시와 뒤쪽 UI 모형도 참고용입니다.

다만 아래 세 묶음만은 본문에 적힌 그대로 맞춰 주시기를 부탁드립니다. 화면에 보이는 값이 시스템이 보낸 값과 어긋나면 현장에서 문제가 생겼을 때 원인을 짚기 어려워지기 때문입니다.

- **값을 가공 없이 그대로 표시**: HW 상태(`hw_state`)·최근 명령(`job_type`)·출발/도착 지점 라벨
- **어휘와 형식**: Unit 식별값 형식(`투입코드_유닛번호`)·Log 분류 3종(`시스템 → AMMR`·`AMMR → 시스템`·`설정 변경`)
- **동작 조건**: 설정 화면 입력 박스와 [일괄 보고 재로드] 버튼의 활성 조건·재로드 응답 3초 대기

이 셋을 뺀 나머지는 모두 업체에서 결정하실 영역입니다.

## 1. 태블릿 UI 개요

AMMR 1대당 태블릿 1대가 본체에 Mount되며, 항상 켜져 있고 한 계정으로 자동 로그인된 상태에서 운영됩니다. 이 태블릿은 담당자가 현장에서 AMMR의 현재 상태를 확인하고 필요한 설정을 조정하기 위한 화면입니다.

이 문서에서는 다음 항목을 제안드립니다.

- 모든 화면 공통으로 항상 보여주셨으면 하는 AMMR 정보 (상단 고정 영역)
- 운영 중 화면 4종 (적재 상태 / 배정 상태 / 설정 / Log)
- 각 화면에 표시되었으면 하는 정보 항목
- 설정 화면의 입력 항목과 동작
- 참고용 UI 모형 4장 (화면별)

---

## 2. 권장 화면 구성

태블릿 화면은 **왼쪽 Sidebar + 본문 영역** 구조로 구성을 권장드립니다.

- **왼쪽 Sidebar**: 화면 전환 메뉴 (적재 상태 / 배정 상태 / 설정 / Log 4개 항목). 모든 화면에서 항상 표시되고, Sidebar 항목을 누르면 본문 영역이 해당 화면으로 전환되는 동작이면 좋겠습니다.
- **상단 정보 영역**: AMMR의 핵심 정보를 표시하는 영역으로, 어느 화면에 있더라도 항상 같은 위치에 고정 표시됩니다.
- **본문 영역**: Sidebar에서 선택된 화면이 표시되는 영역입니다.

화면 디자인·색상·폰트·세부 배치는 업체 자유 결정 영역입니다.

## 3. 상단 정보 영역 (모든 화면 공통, 항상 고정 표시)

다음 7개 항목이 어느 화면에서나 보이도록 구성해 주시면 좋겠습니다. 표기 형식은 업체 자유 결정 영역입니다.

※ 상단 정보 항목

| 항목         | 내용                                                                       |
|--------------|----------------------------------------------------------------------------|
| AMMR ID      | AMMR 고유 식별자 (예: `AMMR-LOGI001`)                                           |
| HW 상태      | AMMR이 보고하는 현재 상태                                                  |
| 최근 명령    | 시스템이 마지막으로 지시한 명령 (job_type)                                 |
| 명령 Slot | 최근 명령이 사용한 이 AMMR의 Slot 번호(1~6) — Pickup=적재 대상 / Dropoff=출고 대상 / Move·Charge=없음(빈값). 명령 이력 있을 때만 표시                  |
| 위치         | 현재 위치 좌표(x, y)와 방향각(a) (예: `12.4, 8.7, 90°`)                    |
| Battery       | 현재 충전량 (%)                                                            |
| 시스템 연결  | 시스템과의 연결 상태 표시 + 연결/해제 동작 버튼                            |

각 항목의 구체 표기 형식·기호·아이콘은 업체 자유 결정 영역입니다. 내용이 식별 가능한 형태로만 구성되면 됩니다. 단, HW 상태·최근 명령 값은 고정 항목입니다 — 아래 표기 값 안내를 참고해 주세요.

**HW 상태·최근 명령 표기 값:**

HW 상태는 AMMR이 자체 보고하는 물리 상태이고, 최근 명령은 시스템이 마지막으로 지시한 Job 명령입니다.

| 항목      | 표기 값                                                              |
|-----------|----------------------------------------------------------------------|
| HW 상태   | `move` / `pickup` / `dropoff` / `charge` / `idle` / `charging` / `low_battery` / `error` (AMMR 자체 보고 hw_state 값) |
| 최근 명령 | `move` / `pickup` / `dropoff` / `charge` (시스템이 지시한 job_type)  |

HW 상태·최근 명령 두 칸은 문서 안내의 고정 항목입니다 — AMMR이 보고한 hw_state 값과 시스템이 지시한 job_type 값을 가공·번역 없이 그대로 표시해 주시기 바랍니다. 시스템이 수신한 값과 화면 표시가 일치해야 현장에서 문제가 생겼을 때 원인을 짚을 수 있습니다.

HW 상태는 AMMR의 현재 물리 동작이고 최근 명령은 시스템이 마지막으로 지시한 명령이라 서로 다를 수 있어 두 값을 따로 표시합니다 (명령 이후 완료·전이하면 HW 상태만 바뀝니다).

**최근 명령·명령 Slot 갱신 시점:** 이 두 값은 Core가 새 Job 지시를 보낼 때 갱신되며, 명령이 완료·전이해도 다음 명령 전까지 마지막 값을 유지합니다 (HW 상태는 실시간, 최근 명령은 명령 단위 갱신). 명령 이력이 없거나 시스템 연결이 끊긴 동안에는 빈값으로 표시됩니다.

**시스템 연결 상태 + 연결/해제 버튼:**

AMMR에 장애·오류가 생겼을 때 담당자가 시스템 연결을 끊고 조치한 뒤 다시 연결할 수 있어야 하므로, 연결/해제 버튼을 함께 제공해 주시기 바랍니다. 연결 중일 때와 끊김 상태일 때를 구분할 수 있는 시각 표시(예: 색상·아이콘·기호 등)와 상태에 따라 동작 가능한 버튼(예: `[연결]` / `[해제]` 또는 Toggle 등)을 함께 두는 형태가 권장됩니다. 구체 표기·버튼 형태는 업체 자유 결정 영역입니다.

'시스템 연결' 상태 판정에는 태블릿 자신의 MQTT 연결뿐 아니라 시스템(Core) 측 연결 상태도 반영합니다 — 태블릿의 MQTT 연결이 살아 있어도 시스템(Core)이 다운되면 시스템 권위 값(최근 명령·명령 Slot·Slot의 Unit 정보·배정)을 받을 수 없으므로 '시스템 연결 끊김'으로 표시합니다. 시스템은 자신의 연결 상태를 알리는 메시지를 발행하며, 태블릿은 이를 구독해 시스템 다운을 인지합니다 (수신 방식·메시지는 ICD를 따릅니다).

**시스템 연결이 끊긴 동안의 표시 방식:**

이 문서에서 다루는 표시값은 출처에 따라 권위가 둘로 나뉩니다 — AMMR/태블릿 자체 값(HW 상태·위치·Battery·slot_state 등 AMMR·태블릿 권위)과 Core가 명령에 담아 내려주는 값(최근 명령·명령 Slot·Slot의 Unit 정보·배정 상태 화면의 표시값 등 시스템 권위). 시스템 연결이 끊긴 동안에는 두 권위 출처에 따라 표시 방식이 달라집니다. AMMR 권위 값은 시스템 연결과 무관하게 마지막 수신값을 그대로 유지하고, 시스템 권위 값은 시스템의 판단이 필요한 값이므로 빈값으로 표시됩니다. 이 일반 원칙은 이 문서의 적재 상태·배정 상태 화면 표시값에 동일하게 적용됩니다.

이 상단 정보 영역에서의 항목별 적용은 다음과 같습니다.

- **HW 상태·위치·Battery**: AMMR 자체에서 보고되는 값이므로 마지막 수신값을 그대로 유지합니다.
- **최근 명령·명령 Slot**: Core가 명령에 담아 주는 값이므로 빈값으로 표시합니다 (예: 항목 칸을 빈 텍스트로 두거나 회색 계열로 표시).

시스템 연결 항목의 끊김 표시(예: ○ + `[연결]`)가 단절 상태를 표면화하는 역할을 합니다. 단절 상태에서 다른 항목 값에 별도의 시각 구분(예: 흐리게·기호 부착 등)을 추가하실지 여부는 업체 자유 결정 영역입니다.

**표시값 갱신 규칙 (모든 화면 공통):**

표시값은 출처에 따라 갱신 주체·시점이 둘로 나뉩니다.

| 축 | 값 | 갱신 시점 | 시스템 연결 끊김 시 |
|----|----|-----------|---------------------|
| 태블릿 자체 판정 | slot_state(점유·비어있음·사용 보류·실패)·HW 상태·위치·Battery | 실시간 (센서·HW 변화 즉시) | 계속 판정·표시 |
| Core 명령 담긴 값 | 최근 명령·명령 Slot·Unit 식별값·모델·버전·Tray·수량·용도·출발/도착·배정 | Core 메시지 수신 시 (Job 지시 + 정합 응답) | 빈값 (Slot 번호 등 고정 항목만) |

담당자가 별도 새로고침 동작을 하지 않아도 위 규칙으로 자동 갱신됩니다.

## 4. 적재 상태 화면

AMMR의 6개 Slot 각각의 현재 적재 상태(slot_state·Unit 정보)를 격자 형태로 보여주는 화면입니다. Slot당 한 칸의 셀이며, 각 셀에는 다음 정보가 함께 표시됩니다.

※ Slot 셀 표시 정보

| 항목       | 내용                                                                  |
|------------|-----------------------------------------------------------------------|
| Slot 번호  | 1 ~ 6                                                                 |
| Slot 상태  | 클라이언트(태블릿+HW)가 판정한 `slot_state` — `occupied`(점유)/`empty`(비점유)/`blocked`(사용 보류)/`job_failed`(작업 실패) |
| 정보       | Slot에 적재된 Unit 식별값(`투입코드_유닛번호`). 비어 있을 때는 비어 있음을 알 수 있는 표기 |

각 항목의 표기 방법은 업체 자유 결정 영역입니다. 예를 들어 slot_state의 `occupied`·`empty`는 센서 표기(`●/○`·`ON/OFF`·아이콘 등), 비어 있음은 `(비어 있음)`·`Empty`·`--` 등 내용이 통하는 어떤 형태로든 표시 가능합니다.

**slot_state 표기 방식:**

slot_state는 태블릿이 판정한 값을 그대로 표시합니다. `occupied`·`empty`는 센서 표기(`●/○` 등)로 두시고, `blocked`(사용 보류)·`job_failed`(작업 실패)는 정상 상태와 한눈에 구분되게 표시해 주시면 좋겠습니다(빨강 계열 권장). 구체 표기와 보조 시각 표시(아이콘 등)는 업체 자유 결정 영역입니다.

**`blocked`(사용 보류)의 의미:**

slot_state가 `blocked`라는 것은 태블릿이 해당 Slot을 사용 보류로 판정했다는 의미입니다. 예를 들어 센서는 점유로 잡혀 있는데 식별된 Unit 정보가 없는 경우(사람이 임의로 Unit을 올려놓았으나 시스템이 지시하지 않은 상황 등)에 발생합니다. 이 경우 센서는 점유(●)이지만 빨강으로 표기되고 정보 칸은 비어 있음을 나타냅니다. `job_failed`는 Job 실패로 해당 Slot이 이상 상태가 된 경우이며 마찬가지로 빨강으로 표기됩니다.

**Unit 식별값과 스티커:**

Unit 식별값은 `투입코드_유닛번호` 형태로 표시됩니다. 예시는 `26SF03002001_001`. 투입코드 부분은 시스템이 외부로부터 받은 발주 단위 코드이고, 언더바 뒤 3자리는 해당 투입코드 안에서 부여된 Unit 일련번호입니다.

Unit 실물에는 이 식별값을 사람이 눈으로 대조할 수 있는 라벨과, 시스템이 기계 입력으로 받을 수 있는 QR 코드가 함께 부착된 스티커가 붙어 있습니다. 라벨에는 이 식별값(`투입코드_유닛번호`)이 사람이 읽을 수 있는 형태로 적혀 있고, QR 코드에는 시스템 권위 식별값인 Unit ID(uuid)가 인코딩되어 있습니다.

이 화면에 표시되는 식별값은 담당자가 현장에서 스티커 라벨과 직접 대조해 확인하기 위한 정보입니다. Unit ID는 적재 정보를 수동으로 입력해야 하는 경우 설정 화면의 적재 정보 관리 영역에서 사용되며, 자세한 동작은 이 문서 뒤쪽 설정 화면과 적재 정보 재로드 동작 안내 부분을 참고해 주세요.

**시스템 연결이 끊긴 동안의 표시 방식:**

표시값은 권위 출처에 따라 단절 동안 표시 방식이 달라집니다.

- **Slot 번호**: 고정 항목이므로 항상 표시됩니다.
- **Slot 상태(slot_state)**: 태블릿이 자체 판정하는 값이므로 단절 동안에도 계속 판정·표시합니다.
- **정보(Unit 식별값)**: Core가 명령에 담아 주는 값이므로 단절 동안 빈값으로 표시합니다.

**갱신 방식:**

slot_state는 태블릿이 자체 판정으로, Unit 정보는 Core가 명령에 담아 준 값으로 자동 갱신됩니다. 담당자가 별도 새로고침 동작을 하지 않아도 됩니다.

## 5. 배정 상태 화면

각 Slot에 매칭된 Job 정보를 보여주는 화면입니다. 6개 Slot 모두 항목으로 표시되며, Job이 배정되어 있지 않거나 Slot이 비어 있는 경우 또는 사용 보류(blocked) 상태인 경우에는 그 상태를 알 수 있는 표기로 표시됩니다.

※ Slot별 표시 정보 (Job이 배정된 Slot에 한함)

| 항목         | 내용                                                                                                       |
|--------------|------------------------------------------------------------------------------------------------------------|
| Unit 식별값  | 해당 Slot에 적재된 Unit 식별값 (적재 상태 화면과 동일 형식 `투입코드_유닛번호`)                            |
| 모델         | Unit에 담긴 제품의 모델 코드                                                                               |
| 버전         | 모델의 버전 표기                                                                                           |
| Tray 단      | Unit을 구성하는 Tray 단 수                                                                                 |
| 수량         | Unit 안 제품 개수                                                                                          |
| 용도         | Unit에 담긴 제품의 용도 (GM 목적 값·별표 등 장식문자 제외)                                                 |
| 출발지       | 이 Unit이 출발한 공정·설비 — 시스템이 명령에 담아 준 출발 라벨을 그대로 표시 (예: `WIP-CLN001`)               |
| 도착지       | 이 Unit이 갈 다음 공정·설비 — 시스템이 명령에 담아 준 도착 라벨을 그대로 표시 (예: `CNC-RAC-A02`)            |

표기 형식·배치는 업체 자유 결정 영역입니다.

**비어 있는 Slot · Job 미배정 Slot · 사용 보류(blocked)의 표기:**

비어 있는 Slot은 비어 있음을 알 수 있는 표기(예: `(비어 있음)`), 적재되어 있으나 Job이 아직 배정되지 않은 Slot은 작업 배정 대기를 알 수 있는 표기(예: `(작업 배정 대기)`), 사용 보류(`blocked`)·작업 실패(`job_failed`)는 그 상태를 알 수 있는 빨강 표기(예: `(사용 보류)`·`(작업 실패)`)가 권장됩니다.

작업 배정 대기 상태는 두 가지 상황에서 발생합니다. 첫째, 적재 정보는 들어왔지만 시스템이 아직 매칭된 Job을 결정하기 전인 일시적인 상태입니다. 둘째, 시스템 연결이 끊긴 동안 Slot의 시스템 권위 정보 일체가 빈값으로 표시되는 상태입니다.

**시스템 연결이 끊긴 동안의 표시 방식:**

이 화면의 표시값(Unit 식별값·모델·버전·Tray·수량·용도·출발·도착) 일체는 모두 Core가 명령에 담아 주는 값이므로, 시스템 연결이 끊긴 동안에는 Slot 번호 외 표시값이 모두 빈값으로 표시됩니다. 빈 칸은 작업 배정 대기 표기로 두어도 좋습니다.

**우선순위 표기에 대하여:**

Job의 우선순위는 시스템 측이 판단해 Slot에 할당합니다. 이 화면은 순번이 아니라 Slot 번호 기준으로 표시되며, 별도의 우선순위 번호는 표시하지 않습니다.

**갱신 방식:**

이 화면의 모든 정보 역시 Core가 명령에 담아 준 값으로 자동 갱신됩니다.

## 6. 설정 화면

설정 화면은 다음 3개 영역으로 구성됩니다 — **Battery·충전 운영**, **시스템 접속**, **적재 정보 관리**.

각 영역의 입력 항목은 영역별 동작 방식으로 시스템 또는 AMMR에 반영됩니다.

- **Battery·충전 운영 영역**: 영역 안 입력 항목을 모은 뒤 [저장] 버튼으로 일괄 반영합니다. AMMR 자체에서 사용하는 값이므로 시스템 연결 상태와 무관하게 항상 활성화됩니다.
- **시스템 접속 영역**: 영역 안 입력 항목을 모은 뒤 [저장] 버튼으로 일괄 반영합니다. 이 영역은 시스템 연결이 끊긴 상태에서만 활성화됩니다 (운영 중 시스템 접속 정보를 변경하면 통신에 영향을 주는 것을 피하기 위함입니다).
- **적재 정보 관리 영역**: Slot별 입력 박스로 Unit ID를 수동 입력하고 [일괄 보고 재로드] 버튼으로 시스템에 일괄 전달하는 방식입니다. 시스템 연결 상태에 따라 입력 박스와 버튼의 활성 방식이 반대로 동작하며, 자세한 내용은 이 영역 항목 설명에서 다룹니다.

이 화면 3개 영역의 입력 값은 모두 AMMR 내부에 저장되어 AMMR 재시작 후에도 유지됩니다. 다만 적재 정보 관리 영역의 Slot 저장 입력값은 재시작 시 센서가 미점유로 판별된 Slot에 한해 비워집니다(아래 적재 정보 관리 참고).

각 영역의 버튼 명칭·형태·배치는 업체 자유 결정 영역입니다.

### 6.1 Battery·충전 운영

※ Battery·충전 운영 입력 항목

| 항목                  | 초기값 | 비고                                                                   |
|-----------------------|--------|------------------------------------------------------------------------|
| 충전 스테이션 번호    | -      | AMMR이 충전 시 이동할 스테이션 번호 (자율 충전·시스템 충전 지시 공통) |
| Battery 대기 임계치   | 80%    | AMMR이 충전을 마쳤다고 판단하는 값                                     |
| Battery 저전력 임계치 | 20%    | AMMR이 자율 충전 진입을 판단하는 값                                    |

충전 스테이션 번호는 단순 정수로 입력받습니다 (예: `1`, `2`, `3`).

**이 영역 안내 문구 (화면에 함께 표시):**

> 충전 스테이션 번호는 AMMR이 충전 시 이동할 스테이션 위치이고(자율 충전과 시스템의 충전 지시 모두 이 스테이션을 사용합니다), Battery 대기·저전력 임계치는 AMMR이 지정된 충전 스테이션으로 스스로 충전·복귀 동작을 결정하는 기준값입니다. 초기값은 각각 80%·20%로 설정되어 있으며, 운영 중 현장 상황에 맞게 설정 화면에서 변경하실 수 있습니다. 변경하신 값은 [저장] 버튼을 누르신 시점에 적용됩니다.

### 6.2 시스템 접속

※ 시스템 접속 입력 항목

| 항목                 | 초기값 | 비고                                                    |
|----------------------|--------|---------------------------------------------------------|
| 시스템 Host IP       | -      | AMMR이 시스템에 접속할 IP 주소                          |
| 시스템 포트          | -      | 시스템 접속 포트 번호                                   |
| MQTT 접속 ID         | -      | 시스템 측 MQTT Broker 접속 자격증명                     |
| MQTT 접속 비밀번호   | -      | 시스템 측 MQTT Broker 접속 자격증명                     |
| 일괄 보고 주기       | 60초   | AMMR이 운영 상태값 전체를 주기적으로 자동 보고하는 간격 |
| 위치(pose) 보고 주기 | 1초    | AMMR이 위치(pose·좌표+방향각)를 자동 보고하는 간격      |
| Battery 보고 주기    | 10초   | AMMR이 Battery 상태를 자동 보고하는 간격                |

위 항목을 입력하신 뒤 [저장] 버튼으로 일괄 적용됩니다. 이 영역 항목은 시스템 연결이 끊긴 상태에서만 변경 가능합니다.

### 6.3 적재 정보 관리

이 영역은 **Slot별 Unit 정보 입력 박스 6개 + Slot별 [초기화] 버튼 6개**와 **일괄 보고 재로드 버튼 1개**로 구성해 주시면 좋겠습니다.

이 영역은 시스템 연결이 끊긴 동안 담당자가 Slot별 적재 정보를 수동으로 입력해 두고, 시스템 재연결 후 [일괄 보고 재로드] 버튼으로 일괄 전달해 사용 보류(blocked)를 풀기 위한 영역입니다. 정상 운영(시스템 연결 유효 + AMMR Job 수행) 중에는 시스템이 명령에 담아 Slot의 적재 정보를 채우므로, 이 영역에 담당자가 별도로 개입할 필요가 없습니다 (자세한 동작은 이 문서 뒤쪽 적재 정보 재로드 동작 안내 부분을 참고해 주세요).

**Slot별 Unit 정보 입력 박스:**

- Slot 1 ~ 6 각각에 텍스트 입력 박스를 1개씩 배치해 주시면 좋겠습니다.
- 이 입력 박스에 들어가는 값은 **Unit ID** (시스템이 부여하는 시스템 권위 QR 식별값·uuid)입니다. 이 Unit ID는 적재 상태 화면에 표시되는 `투입코드_유닛번호` 라벨과는 별도의 값입니다. 라벨은 담당자가 Slot의 Unit을 시각적으로 식별하기 위한 사람 읽기용 표기이고, Unit ID는 시스템이 Unit을 권위적으로 인지하기 위한 시스템 입력용 식별값입니다.
- 적재된 Slot의 입력 박스에는 마지막 시스템 응답 또는 [재로드] 시점의 Unit ID가 기본값으로 표시됩니다. 이 값은 AMMR 내부에 저장되어 재시작 후에도 유지되며(재시작 시 센서 판별에서 미점유로 나온 Slot은 저장값을 비웁니다 — 아래 규칙), 정상 운영 중 명령 선탑재 값으로 갱신되는 곳에는 갱신값으로 덮어 씌워집니다. 비어 있는 Slot의 입력 박스는 빈 상태로 표시됩니다.
- 입력 박스를 터치하면 활성화되어 가상 키보드가 표시되고, 담당자가 직접 값을 입력할 수 있습니다.
- 향후 QR 리딩을 통한 입력도 지원되도록 구성해 주시면 좋겠습니다 (스티커에 부착된 QR 코드를 스캐너로 읽어 해당 Slot의 입력 박스 값으로 채워 넣는 방식. QR 코드에는 이 Unit ID(uuid)가 인코딩되어 있습니다).

**Slot별 [초기화] 버튼:**

- 각 Slot 입력 박스 우측에 [초기화] 버튼을 1개씩 배치해 주시면 좋겠습니다.
- 이 버튼은 해당 Slot 입력 박스의 값을 비우는 동작입니다. AMMR 내부 저장값도 즉시 비워집니다.
- 시스템에 단독 전달되지는 않습니다 — 입력 박스 값 변경분은 [일괄 보고 재로드] 버튼을 눌러야 시스템에 일괄 전달됩니다.

**일괄 보고 재로드 버튼:**

이 버튼은 AMMR이 6Slot 전체의 slot_state + 입력된 Unit ID(있으면)를 시스템에 일괄 전달하는 동작을 Trigger합니다. 시스템은 받은 정보를 Core 마스터 배정과 대조해 Slot별 Core 확정 Unit 정보·Job 배정을 일괄보고 응답(state/reconcile)으로 내려주며(재로드는 정합이 일치해도 응답이 옵니다), 이 응답으로 적재 상태 화면과 배정 상태 화면이 일괄 정정됩니다.

**이 영역 안내 문구 (화면에 함께 표시):**

> Slot별 입력 박스를 터치하면 가상 키보드가 표시되어 값을 직접 입력하실 수 있습니다. 향후 QR 리딩으로도 입력 가능합니다. Slot별 [초기화] 버튼을 누르면 해당 Slot 값이 비워집니다. [일괄 보고 재로드] 버튼을 누르시면 6Slot 전체의 slot_state와 입력된 Unit ID가 시스템에 일괄 전달되어 사용 보류(blocked)가 자연스럽게 풀립니다. 입력 박스와 [초기화] 버튼은 시스템 연결이 끊긴 상태에서만, [재로드] 버튼은 시스템 연결이 유효한 동안에만 활성화됩니다. 이 화면 입력 값은 AMMR 재시작 후에도 유지되나, 재시작 시 센서가 미점유로 판별된 Slot은 저장값이 비워집니다. 자세한 동작은 이 문서 뒤쪽 적재 정보 재로드 동작 안내 부분을 참고해 주세요.

**활성화 조건:**

이 영역의 활성화 방식은 입력 박스·[초기화] 버튼과 [일괄 보고 재로드] 버튼이 서로 반대입니다.

- **Slot별 입력 박스·[초기화] 버튼**: 시스템 연결이 끊긴 상태에서만 활성화됩니다. 정상 운영 시에는 시스템이 적재 정보를 자동으로 채우므로 수동 입력이 불필요합니다.
- **[일괄 보고 재로드] 버튼**: 시스템 연결이 유효한 동안에만 활성화됩니다. 시스템에 정보를 전달하는 동작이라 연결이 끊긴 동안에는 동작할 수 없습니다.

따라서 일반적인 사용 흐름은 다음과 같습니다 — 시스템 연결이 끊긴 동안 Slot 입력 박스에 Unit ID를 입력해 두고 → 시스템 재연결. 재연결 시점의 일괄 보고(초기 연결·주기 자동 발행)에 입력해 둔 값이 자동으로 실려 전달되므로 [일괄 보고 재로드]를 누르지 않아도 반영되며, [일괄 보고 재로드] 버튼은 이를 즉시 한 번 발행하는 동작입니다.

**미점유 Slot 값 비움 (센서 판별):** 센서 판별에서 미점유(empty)로 나온 Slot은 그 Slot 관련 표시값 일체(적재 상태 화면 정보·배정 상태 화면 표시값·이 영역의 저장 입력값)를 비웁니다. 이 판별은 운영 중 센서 변화 시점과 AMMR 재시작(부팅) 시 센서 판별 시점 모두에 적용됩니다. 저장 입력값이 재시작 후에도 유지되는 것은 점유(occupied) Slot에 한하며, 그 값이 재연결 시 일괄 보고(스냅샷)에 실려 일괄보고 응답으로 정합됩니다.

## 7. 적재 정보 재로드 동작 안내

이 장은 적재 정보가 시스템과 AMMR 사이에서 어떻게 흐르는지, 그리고 담당자가 수동으로 개입해야 하는 상황에서 [일괄 보고 재로드] 버튼이 어떻게 동작하는지를 정리했습니다.

**정상 운영 시 적재 정보 흐름:**

정상 운영(시스템 연결 유효 + AMMR Job 수행) 중에는 AMMR Slot의 적재 정보를 시스템이 자동으로 채워 줍니다. AMMR이 Pickup 또는 Dropoff 동작을 수행하면 Slot의 slot_state 변동이 시스템에 보고됩니다. Unit 정보(식별값·모델·버전·Tray·수량)와 Job은 시스템이 Pickup/Dropoff 명령에 미리 담아 주므로 태블릿이 자체 채웁니다. 담당자가 설정 화면의 적재 정보 관리 영역에 별도로 개입할 필요는 없습니다.

**수동 입력이 필요한 상황:**

시스템 연결이 끊긴 상태에서 담당자가 현장에서 AMMR Slot의 Unit을 임의로 가져가거나 새로 올려놓는 상황이 발생할 수 있습니다. 시스템 측은 단절 동안 AMMR Slot의 변동을 인지하지 못하므로, 재연결 직후 적재된 Slot을 사용 보류(blocked) 상태로 표시해 둡니다. 담당자는 시스템 연결이 끊긴 동안 사용 보류를 미리 풀어 두기 위해 설정 화면의 적재 정보 관리 영역에서 Slot별 입력 박스에 Unit ID를 직접 입력합니다. Unit ID 입력은 가상 키보드 또는 QR 리딩(스티커 QR 코드에 인코딩된 Unit ID를 스캐너로 읽어 입력)으로 가능합니다.

**[일괄 보고 재로드] 일괄 전달 동작:**

담당자가 [일괄 보고 재로드] 버튼을 누르면 AMMR은 6Slot 전체의 slot_state + 입력된 Unit ID(있으면)를 시스템에 일괄 전달합니다. 시스템은 Core 마스터 배정으로 확정한 Slot별 응답(state/reconcile)을 일괄 내려줍니다. 응답에는 Slot별 Core 확정 Unit 정보(식별값·모델·버전·Tray·수량) + Job 배정 정보(출발·도착)가 포함되며, 이 응답으로 적재 상태 화면과 배정 상태 화면이 일괄 정정됩니다. 이 일괄 전달은 별도 재로드 전용 통신이 아니라 일괄 보고(초기 연결·주기 자동 발행과 같은 메시지)를 수동으로 한 번 발행하는 것이며, 발행 계기는 메시지의 `trigger` 값으로 구분되어 재로드는 `manual`로 실립니다. 끊긴 동안 입력해 둔 값은 재연결 시점의 일괄 보고에 자동으로 실리므로, [일괄 보고 재로드]를 누르지 않아도 반영됩니다.

**응답 없음 시 처리:**

[일괄 보고 재로드] 버튼을 누른 뒤 3초 안에 시스템 응답이 오지 않으면 실패로 판단해 메시지 박스로 안내하고, 담당자가 다시 시도하실 수 있습니다.

**버튼 활성화 조건:**

[일괄 보고 재로드] 버튼은 시스템 연결이 유효한 동안에만 활성화됩니다. 시스템 연결이 끊긴 상태에서는 비활성 표시됩니다 (시스템에 정보를 전달하는 동작이라 연결이 끊긴 동안에는 동작할 수 없습니다). 반대로 Slot별 Unit 정보 입력 박스는 시스템 연결이 끊긴 상태에서만 활성화됩니다 (정상 운영 시에는 시스템이 자동으로 채우므로 수동 입력이 불필요합니다).

**응답 종류:**

시스템 응답은 Slot별로 다음 중 하나입니다.

- **정상 응답**: Core 확정 Unit 정보·Job 배정이 응답되어 화면에 표시됩니다.
- **빈 Slot 응답**: slot_state가 `empty`인 Slot은 빈 Slot으로 응답되어 적재 상태 화면의 정보 칸이 비어 있음으로 표시됩니다.
- **사용 보류(blocked) 응답**: 시스템이 입력된 Unit ID를 인지하지 못하거나, 적재된 Unit이 갈 수 없는 공정으로 들어온 것으로 판단되면 해당 Slot을 확정하지 못하며, 태블릿은 그 Slot을 `blocked`(사용 보류)로 표시합니다.

`blocked`(사용 보류)로 남은 Slot은 담당자가 입력 박스 값을 재확인하거나 Slot 실물을 확인하여 다시 [재로드]를 시도할 수 있습니다.

**AMMR 단절 중 Unit 바꿔치기:** 시스템 연결이 끊긴 동안 Slot의 Unit이 다른 Unit으로 교체되면, 센서는 계속 점유로 잡혀 자동 감지되지 않습니다. 이 경우 담당자가 적재 정보 관리 영역에서 해당 Slot의 Unit ID를 새 Unit 값으로 재입력(재맵핑)해야 합니다(현재 수동 입력, 향후 QR 리딩 지원). 재맵핑하지 않으면 옛 Unit ID가 그대로 보고되어 오배송이 발생할 수 있으며, 이때는 목적지 공정의 QR 확인 단계에서 불일치가 검출되어 처리됩니다. AMMR 자체에는 Slot Unit 재판독 수단(QR)이 없어 이 재맵핑은 담당자 책임이며, 향후 AMMR QR 도입 시 자동화됩니다.

## 8. Log 화면

AMMR에서 발생한 주요 Event를 시간순으로 보여주는 화면입니다. 이 화면의 Log는 AMMR이 자체적으로 기록한 것으로, 시스템 측은 이 화면에 개입하지 않습니다. 담당자가 현장에서 발생한 상황을 점검할 수 있도록 구성해 주시면 좋겠습니다.

**기록 분류:**

다음 3종 분류를 한 화면에 통합 시간순으로 표시해 주시면 좋겠습니다. 분류는 시스템과 주고받은 방향을 기준으로 나눴습니다. 분류 어휘는 문서 안내의 고정 항목이라, 본문에 적힌 그대로 적용해 주시기 바랍니다.

- **시스템 → AMMR**: 시스템에서 받은 Event (작업 지시 수신, 일괄 보고 응답 수신, 연결 상태 수신 등)
- **AMMR → 시스템**: 시스템으로 보낸 Event (일괄 보고 발행, 작업 결과 보고, 지시 수신 확인 발신, HW 상태 전이·Slot 적재 정보 변동 보고 등)
- **설정 변경**: 설정 화면에서 사용자가 변경한 항목 (임계치 변경, 접속 정보 변경 등)

**표시 컬럼:**

| 컬럼 | 내용                                                          |
|------|---------------------------------------------------------------|
| 시간 | Event 발생 시각 (날짜·시각 표기 형식은 업체 자유 결정 영역)  |
| 분류 | 위 3종 분류 중 하나 (시스템 → AMMR / AMMR → 시스템 / 설정 변경) |
| 내용 | Event 요약 문구                                              |

**분류 필터:**

화면 상단에 분류 필터(전체 / 시스템 → AMMR / AMMR → 시스템 / 설정 변경)를 제공해 주시면 좋겠습니다. 선택된 분류에 해당하는 항목만 리스트에 표시됩니다. 필터 UI 형태(버튼·Dropdown·Toggle 등)는 업체 자유 결정 영역입니다.

**갱신 방식:**

이 화면의 모든 정보는 AMMR이 자체적으로 기록한 Event를 시간순으로 자동 표시합니다.

## 9. 참고용 UI 모형

아래 모형 4장은 참고용입니다. 어떤 정보가 어떤 영역에 배치되면 좋을지 보여드리기 위해 박스 구성과 Slot·표 배치를 시각화했습니다. **실제 화면의 디자인·색상·폰트·세부 Component는 업체 자유 결정 영역**입니다. 모형 안의 표기 예시(상태 문구·라벨 형식·Unit ID 예시 등)도 모두 참고용입니다.

각 모형은 외곽 박스 안에 세 영역(상단 정보 영역 / Sidebar / 본문)으로 분리해서 표시했습니다. 실제 화면에서는 상단 정보 영역이 화면 상단에 고정되고, Sidebar가 화면 좌측에 고정되며, 본문이 나머지 영역에 배치됩니다. Sidebar의 `>` 표시는 현재 선택된 화면을 가리키는 표시 예시입니다.

본문 길이가 한 화면을 넘기면(특히 설정 화면) 본문 영역 내부 스크롤로 보이는 방식이면 좋겠습니다.

이 모형들은 ASCII·박스 문자(─│┌)·기호(●○·°)를 1폭으로, 한글을 2폭으로 두는 가시폭 가정으로 정렬되어 있습니다. D2Coding 같은 한글 Monospace 폰트로 보시면 박스가 정확히 정합되어 보입니다. 그 외 폰트에서는 일부 기호·박스 문자의 폭이 달라 Slot 박스 모서리가 미세하게 어긋나 보일 수 있으나, 정보 배치 의도는 동일합니다.

모형 안 `상태 ●` 표기는 색으로 slot_state를 구분합니다. `occupied`·`empty`는 상단 정보 영역의 시스템 연결 표기와 같은 색으로, `blocked`·`job_failed` 시는 빨강 계열이면 좋겠습니다. 이 모형은 ASCII 표기이므로 색 표기는 없습니다.

```
└───────────────┴───────────────────────────────────────────────────────────────────────────────────────────────┘
```

이 모형은 시스템 연결이 유효한 상태(상단 시스템 연결 ●)를 가정해 그렸습니다. 이 가정에 따라 영역별 활성 부분은 다음과 같습니다.

- **Battery·충전 운영 영역**: AMMR 자체 값이므로 시스템 연결 상태와 무관하게 항상 활성. 모형 상태에서도 활성.
- **시스템 접속 영역**: 이 영역 항목은 시스템 연결이 끊긴 상태에서만 활성이라 모형 상태(연결 유효)에서는 비활성.
- **적재 정보 관리 영역**: Slot별 입력 박스·[초기화] 버튼은 시스템 연결이 끊긴 상태에서만 활성이라 모형 상태에서는 비활성, [일괄 보고 재로드] 버튼은 시스템 연결이 유효한 동안만 활성이라 모형 상태에서는 활성.
